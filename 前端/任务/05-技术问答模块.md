# 模块05：技术问答模块

> 重要：本模块的接口路径必须与后端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
构建类似StackOverflow的技术问答系统，支持提问、回答、投票、采纳最佳答案、悬赏等功能。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 问题状态标识清晰（已解决/未解决/悬赏中）
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 问题列表
- 分类筛选（全部、未解决、已解决、悬赏中）
- 标签筛选
- 排序（最新、热门、未回答、悬赏）
- 问题卡片展示（标题、摘要、标签、投票数、回答数、浏览数、状态标识）

### 2. 提问功能
- 标题输入（智能提示相似问题，避免重复提问）
- Markdown内容编辑器
- 标签选择
- 悬赏积分设置（可选）
- 代码块插入
- 发布前预览

### 3. 问题详情页
- 问题完整内容
- 投票（赞同/反对）
- 回答列表
  - 按投票数排序
  - 最佳答案置顶高亮
  - 每个回答可投票
- 提问者采纳最佳答案
- 评论区（问题评论 + 回答评论）
- 关注问题（有新回答时通知）
- 编辑历史记录

### 4. 回答功能
- Markdown编辑器回答
- 代码块支持
- 回答预览

### 5. 悬赏系统
- 设置悬赏积分
- 悬赏倒计时
- 到期自动结算或手动采纳

## 组件结构
```
src/pages/QA/
├── QuestionList.tsx          # 问题列表
├── QuestionDetail.tsx        # 问题详情
├── AskQuestion.tsx           # 提问页面
├── EditQuestion.tsx          # 编辑问题
└── components/
    ├── QuestionCard.tsx       # 问题卡片
    ├── AnswerItem.tsx         # 回答项
    ├── VoteButton.tsx         # 投票组件
    ├── BountyBadge.tsx        # 悬赏标识
    ├── StatusBadge.tsx        # 状态标识
    ├── SimilarQuestions.tsx   # 相似问题提示
    └── AcceptButton.tsx       # 采纳按钮
```

## 接口对接
```
GET    /api/v1/questions              # 问题列表
GET    /api/v1/questions/:id          # 问题详情
POST   /api/v1/questions              # 创建问题
PUT    /api/v1/questions/:id          # 编辑问题
DELETE /api/v1/questions/:id          # 删除问题
POST   /api/v1/questions/:id/vote     # 投票
POST   /api/v1/questions/:id/answers  # 提交回答
PUT    /api/v1/answers/:id            # 编辑回答
POST   /api/v1/answers/:id/accept     # 采纳最佳答案
POST   /api/v1/answers/:id/vote       # 回答投票
POST   /api/v1/questions/:id/bounty   # 设置悬赏
GET    /api/v1/questions/similar      # 查询相似问题
```

## 验收标准
- 提问时能智能推荐相似问题
- 投票和采纳操作实时响应
- 问题状态标识准确
- 悬赏系统逻辑正确
- 回答排序合理（最佳答案置顶）

# 模块10：消息通知模块

> 重要：本模块的接口路径必须与后端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实时消息通知系统，包括站内通知、系统公告、互动提醒等，基于WebSocket实现实时推送。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 通知弹出动画流畅
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 通知类型
- 互动通知：评论、回复、点赞、收藏、关注
- 系统通知：系统公告、账号安全提醒
- 问答通知：问题被回答、回答被采纳、悬赏到期
- 内容通知：关注的人发布新内容
- 私信通知：用户之间一对一私信提醒

### 2. 通知展示
- 导航栏通知图标 + 未读数量红点
- 通知下拉面板（快速预览最近通知）
- 通知中心页面（完整通知列表）
- 通知分类标签页筛选
- 已读/未读状态标识
- 通知时间智能展示（刚刚、5分钟前、昨天、具体日期）

### 3. 通知操作
- 单条标记已读
- 全部标记已读
- 删除通知
- 点击通知跳转到对应内容

### 4. 实时推送
- WebSocket连接管理
- 新通知实时弹出提示（Toast通知）
- 断线重连机制
- 浏览器通知授权（Web Notification API）

### 5. 通知设置
- 各类通知开关
- 邮件通知开关
- 免打扰时间段设置

## 接口对接
```
GET    /api/v1/notifications              # 通知列表
GET    /api/v1/notifications/unread-count # 未读数量
PUT    /api/v1/notifications/:id/read     # 标记已读
PUT    /api/v1/notifications/read-all     # 全部已读
DELETE /api/v1/notifications/:id          # 删除通知
PUT    /api/v1/notifications/settings     # 更新通知设置

GET    /api/v1/messages/conversations              # 私信会话列表
POST   /api/v1/messages/conversations/open         # 打开会话
GET    /api/v1/messages/conversations/:id/messages # 会话消息
POST   /api/v1/messages/send                       # 发送私信（支持文本+attachments）
POST   /api/v1/messages/:id/recall                # 撤回消息
PUT    /api/v1/messages/conversations/:id/read     # 标记会话已读
GET    /api/v1/messages/unread-count               # 私信未读数量
GET    /api/v1/messages/block/:userId/status       # 屏蔽状态
POST   /api/v1/messages/block/:userId              # 屏蔽用户
DELETE /api/v1/messages/block/:userId              # 取消屏蔽

POST   /api/v1/upload/image                        # 上传图片附件
POST   /api/v1/upload/file                         # 上传文件附件

WebSocket Events:
- connect                              # 连接
- notification:new                     # 新通知
- notification:read                    # 已读同步
- private-message:new                  # 新私信
- private-message:count                # 私信未读数量同步
- private-message:typing               # 正在输入状态
- private-message:recalled             # 消息撤回状态
```

## 验收标准
- WebSocket连接稳定，断线自动重连
- 新通知实时到达，延迟不超过1秒
- 未读数量实时更新
- 通知跳转目标页面准确
- 批量操作响应快速

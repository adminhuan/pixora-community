# 模块08：评论与举报管理模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
管理全站评论和用户举报，处理违规内容，维护社区秩序。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 举报处理流程清晰高效
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 评论管理
- 评论列表（内容、作者、所属内容、时间、状态）
- 搜索筛选（关键词、作者、时间、状态）
- 删除评论
- 批量删除
- 查看评论上下文（定位到原帖）

### 2. 举报管理
- 举报列表（举报内容、举报原因、举报人、被举报人、时间、状态）
- 举报分类（垃圾内容、辱骂、广告、色情、侵权等）
- 举报处理操作
  - 删除被举报内容
  - 警告被举报用户
  - 禁言被举报用户
  - 封禁被举报用户
  - 驳回举报（误报）
- 举报处理结果通知双方
- 举报统计（各类型数量、处理率）

### 3. 黑名单管理
- IP黑名单
- 设备黑名单
- 黑名单添加/移除

## 组件结构
```
src/pages/CommentAdmin/
├── CommentManage.tsx          # 评论管理
├── ReportManage.tsx           # 举报管理
├── BlacklistManage.tsx        # 黑名单管理
└── components/
    ├── CommentTable.tsx        # 评论表格
    ├── ReportTable.tsx         # 举报表格
    ├── ReportDetail.tsx        # 举报详情
    ├── HandleReportModal.tsx   # 举报处理弹窗
    └── BlacklistTable.tsx      # 黑名单表格
```

## 接口对接
```
GET    /api/v1/admin/comments             # 评论列表
DELETE /api/v1/admin/comments/:id         # 删除评论
POST   /api/v1/admin/comments/batch-delete # 批量删除
GET    /api/v1/admin/reports              # 举报列表
PUT    /api/v1/admin/reports/:id/handle   # 处理举报
GET    /api/v1/admin/reports/stats        # 举报统计
GET    /api/v1/admin/blacklist            # 黑名单列表
POST   /api/v1/admin/blacklist            # 添加黑名单
DELETE /api/v1/admin/blacklist/:id        # 移除黑名单
```

## 验收标准
- 举报处理流程完整
- 处理结果通知双方用户
- 批量操作功能稳定
- 黑名单生效即时
- 举报统计数据准确

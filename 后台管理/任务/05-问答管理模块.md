# 模块05：问答管理模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
管理问答系统的问题和回答，处理悬赏、标记重复问题等。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 问题管理
- 问题列表（标题、提问者、状态、回答数、投票数、时间）
- 搜索筛选（关键词、状态、时间、标签）
- 关闭问题
- 标记重复问题（关联已有问题）
- 删除问题
- 编辑问题标签

### 2. 回答管理
- 回答列表查看
- 删除违规回答
- 手动设置最佳答案

### 3. 悬赏管理
- 悬赏问题列表
- 悬赏到期处理
- 手动结算悬赏

## 组件结构
```
src/pages/QAAdmin/
├── QuestionManage.tsx         # 问题管理
├── AnswerManage.tsx           # 回答管理
├── BountyManage.tsx           # 悬赏管理
└── components/
    ├── QuestionTable.tsx       # 问题表格
    ├── DuplicateModal.tsx      # 标记重复弹窗
    └── BountySettleModal.tsx   # 悬赏结算弹窗
```

## 接口对接
```
GET    /api/v1/admin/questions            # 问题列表
PUT    /api/v1/admin/questions/:id/close  # 关闭问题
PUT    /api/v1/admin/questions/:id/duplicate # 标记重复
DELETE /api/v1/admin/questions/:id        # 删除问题
DELETE /api/v1/admin/answers/:id          # 删除回答
PUT    /api/v1/admin/answers/:id/accept   # 设置最佳答案
GET    /api/v1/admin/bounties             # 悬赏列表
POST   /api/v1/admin/bounties/:id/settle  # 结算悬赏
```

## 验收标准
- 问题状态管理逻辑正确
- 标记重复问题关联准确
- 悬赏结算金额计算正确
- 管理操作有确认提示

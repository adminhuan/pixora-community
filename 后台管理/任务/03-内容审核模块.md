# 模块03：内容审核模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
对用户发布的所有内容进行审核管理，包括帖子、博客、代码分享、项目、评论等的审核、删除、恢复操作。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 审核操作流程高效，支持快捷键操作
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 审核队列
- 待审核内容列表（按类型分标签页）
- 审核优先级标识（举报数越多优先级越高）
- 内容预览（不跳转页面直接预览）
- 快速审核操作（通过/拒绝/标记）
- 批量审核

### 2. 审核操作
- 通过审核
- 拒绝并说明原因（选择预设原因或自定义）
- 标记为需要修改（通知作者修改）
- 删除内容
- 恢复已删除内容
- 设为精华/推荐

### 3. 敏感词过滤
- 敏感词库管理（添加/删除/导入）
- 命中敏感词自动标记待审核
- 敏感词匹配记录

### 4. 审核记录
- 审核历史列表
- 按审核员筛选
- 按操作类型筛选
- 审核统计（通过率、处理速度等）

## 组件结构
```
src/pages/Audit/
├── AuditQueue.tsx             # 审核队列
├── AuditHistory.tsx           # 审核历史
├── SensitiveWords.tsx         # 敏感词管理
└── components/
    ├── AuditCard.tsx           # 审核内容卡片
    ├── ContentPreview.tsx      # 内容预览
    ├── AuditActions.tsx        # 审核操作按钮
    ├── RejectReasonModal.tsx   # 拒绝原因弹窗
    └── AuditStats.tsx          # 审核统计
```

## 接口对接
```
GET    /api/v1/admin/audit/queue          # 审核队列
POST   /api/v1/admin/audit/:id/approve    # 通过
POST   /api/v1/admin/audit/:id/reject     # 拒绝
POST   /api/v1/admin/audit/:id/delete     # 删除
POST   /api/v1/admin/audit/:id/restore    # 恢复
POST   /api/v1/admin/audit/batch          # 批量审核
GET    /api/v1/admin/audit/history        # 审核历史
GET    /api/v1/admin/sensitive-words      # 敏感词列表
POST   /api/v1/admin/sensitive-words      # 添加敏感词
DELETE /api/v1/admin/sensitive-words/:id  # 删除敏感词
POST   /api/v1/admin/sensitive-words/import # 导入敏感词
```

## 验收标准
- 审核队列实时更新
- 内容预览加载快速
- 批量审核操作稳定
- 敏感词过滤准确
- 审核记录完整可追溯

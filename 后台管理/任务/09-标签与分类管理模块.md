# 模块09：标签与分类管理模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
管理全站的标签和分类体系，包括标签合并、同义词、分类层级等。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 标签管理
- 标签列表（名称、使用次数、创建时间、状态）
- 搜索标签
- 创建/编辑/删除标签
- 标签合并（将多个同义标签合并为一个）
- 标签别名设置
- 标签描述编辑
- 标签图标设置
- 批量操作

### 2. 分类管理
- 全站分类树形结构管理
- 一级/二级分类的增删改
- 分类排序
- 分类启用/禁用
- 分类关联标签

### 3. 标签统计
- 标签使用趋势
- 热门标签排行
- 标签增长分析

### 4. 分类申请审核
- 分类申请列表（按状态筛选）
- 查看申请人、申请理由、申请时间
- 审核通过/驳回（支持备注）
- 审核记录追踪（审核人、审核时间）

## 接口对接
```
GET    /api/v1/admin/tags                 # 标签列表
POST   /api/v1/admin/tags                 # 创建标签
PUT    /api/v1/admin/tags/:id             # 编辑标签
DELETE /api/v1/admin/tags/:id             # 删除标签
POST   /api/v1/admin/tags/merge           # 合并标签
GET    /api/v1/admin/tags/stats           # 标签统计
GET    /api/v1/admin/categories                        # 分类树
POST   /api/v1/admin/categories                        # 创建分类
PUT    /api/v1/admin/categories/:id                    # 编辑分类
DELETE /api/v1/admin/categories/:id                    # 删除分类
PUT    /api/v1/admin/categories/sort                   # 分类排序
GET    /api/v1/admin/categories/applications           # 分类申请列表
PUT    /api/v1/admin/categories/applications/:id/review # 审核分类申请
```

## 验收标准
- 标签合并后旧标签内容自动迁移
- 分类树操作直观
- 删除有关联内容的标签/分类时有警告
- 标签统计数据准确

# 模块02：用户管理模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
管理员对用户进行查看、搜索、编辑、禁言、封禁等管理操作。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 表格操作直观，批量操作便捷
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 用户列表
- 用户表格（头像、用户名、邮箱、注册时间、等级、状态、操作）
- 搜索筛选（用户名、邮箱、注册时间范围、状态、等级）
- 分页与每页条数选择
- 批量操作（批量禁言、批量封禁）
- 数据导出（CSV/Excel）

### 2. 用户详情
- 基本信息查看
- 发布内容统计
- 积分与等级信息
- 登录日志
- 操作历史
- 举报记录（被举报/举报他人）

### 3. 用户管理操作
- 编辑用户信息
- 修改用户角色（普通用户/版主/管理员）
- 禁言（设置时长）
- 封禁账号（设置时长或永久）
- 解除禁言/封禁
- 重置密码
- 手动调整积分

### 4. 角色权限管理
- 角色列表（管理员、版主、普通用户）
- 角色权限配置
- 分配角色

## 组件结构
```
src/pages/Users/
├── UserList.tsx               # 用户列表
├── UserDetail.tsx             # 用户详情
├── UserEdit.tsx               # 编辑用户
├── RoleManagement.tsx         # 角色管理
└── components/
    ├── UserTable.tsx           # 用户表格
    ├── UserFilters.tsx         # 搜索筛选
    ├── BanModal.tsx            # 封禁弹窗
    ├── MuteModal.tsx           # 禁言弹窗
    └── RoleSelect.tsx          # 角色选择
```

## 接口对接
```
GET    /api/v1/admin/users                # 用户列表（分页、筛选）
GET    /api/v1/admin/users/:id            # 用户详情
PUT    /api/v1/admin/users/:id            # 编辑用户
POST   /api/v1/admin/users/:id/ban        # 封禁
POST   /api/v1/admin/users/:id/unban      # 解封
POST   /api/v1/admin/users/:id/mute       # 禁言
POST   /api/v1/admin/users/:id/unmute     # 解除禁言
PUT    /api/v1/admin/users/:id/role       # 修改角色
POST   /api/v1/admin/users/:id/reset-pwd  # 重置密码
GET    /api/v1/admin/roles                # 角色列表
PUT    /api/v1/admin/roles/:id            # 编辑角色权限
```

## 验收标准
- 用户列表加载速度快，支持大数据量分页
- 搜索筛选响应迅速
- 封禁/禁言操作有确认弹窗
- 操作日志记录完整
- 批量操作功能正常

# 模块09：评论互动模块

> 重要：本模块的接口路径必须与后端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
通用评论系统组件，支持多级嵌套评论、点赞、回复、举报等功能，可复用于帖子、博客、项目、代码分享等场景。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 评论层级关系清晰，嵌套不超过3层
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 评论发表
- Markdown简易编辑器
- @提及用户（搜索 + 选择）
- 代码块插入
- 图片插入
- 评论字数限制提示

### 2. 评论列表展示
- 按时间排序（最新/最早）
- 按热度排序
- 嵌套回复展示（最多3层）
- 展开/折叠长评论
- 展开/折叠回复列表
- 评论总数显示

### 3. 评论互动
- 点赞/取消点赞
- 回复评论
- 删除自己的评论
- 举报评论
- 作者标识（原作者回复有特殊标识）

### 4. 评论管理
- 置顶评论（作者/管理员权限）
- 删除评论（作者/管理员权限）

## 组件结构
```
src/components/Comment/
├── CommentSection.tsx         # 评论区容器
├── CommentInput.tsx           # 评论输入框
├── CommentList.tsx            # 评论列表
├── CommentItem.tsx            # 单条评论
├── ReplyList.tsx              # 回复列表
├── MentionSelect.tsx          # @提及选择器
└── ReportModal.tsx            # 举报弹窗
```

## 接口对接
```
GET    /api/v1/comments?target_type=post&target_id=xxx  # 获取评论列表
POST   /api/v1/comments                                  # 发表评论
PUT    /api/v1/comments/:id                              # 编辑评论
DELETE /api/v1/comments/:id                              # 删除评论
POST   /api/v1/comments/:id/like                         # 点赞评论
POST   /api/v1/comments/:id/report                       # 举报评论
POST   /api/v1/comments/:id/pin                          # 置顶评论
```

## 验收标准
- 评论组件可在多个页面复用
- 嵌套回复层级清晰
- @提及搜索流畅
- 点赞操作无延迟
- 长评论折叠/展开交互自然

# 模块02：用户认证系统

> 重要：本模块的接口路径必须与后端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现完整的用户注册、登录、密码找回、第三方登录等认证功能，以及Token管理和权限控制。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色），登录/注册页面需要精心设计
- 调用 UI/UX PRO MAX skills 设计认证页面
- 表单验证实时反馈，用户体验友好
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 注册功能
- 邮箱注册（邮箱 + 用户名 + 密码）
- 手机号注册（手机号 + 验证码 + 密码）
- 表单实时校验（邮箱格式、密码强度、用户名唯一性）
- 图形验证码
- 用户协议勾选确认

### 2. 登录功能
- 邮箱/用户名 + 密码登录
- 手机号 + 验证码登录
- 记住登录状态
- 第三方登录（GitHub、微信、QQ）

### 3. 密码管理
- 忘记密码 -> 邮箱验证 -> 重置密码
- 登录后修改密码

### 4. Token管理
- JWT Token 自动刷新
- 登录状态持久化（localStorage / httpOnly cookie）
- 多设备登录管理
- 退出登录清除Token

### 5. 路由守卫
- 未登录用户访问受保护页面自动跳转登录
- 已登录用户访问登录页自动跳转首页
- 权限不足提示

## 组件结构
```
src/pages/Auth/
├── Login.tsx              # 登录页
├── Register.tsx           # 注册页
├── ForgotPassword.tsx     # 忘记密码
├── ResetPassword.tsx      # 重置密码
└── OAuthCallback.tsx      # 第三方登录回调

src/hooks/
├── useAuth.ts             # 认证状态Hook
└── usePermission.ts       # 权限判断Hook

src/store/
└── authStore.ts           # 认证状态管理

src/utils/
├── token.ts               # Token工具函数
└── authGuard.tsx          # 路由守卫组件

src/api/
└── auth.ts                # 认证相关API
```

## 接口对接
```
POST /api/v1/auth/register          # 注册
POST /api/v1/auth/login             # 登录
POST /api/v1/auth/logout            # 退出
POST /api/v1/auth/refresh-token     # 刷新Token
POST /api/v1/auth/forgot-password   # 忘记密码
POST /api/v1/auth/reset-password    # 重置密码
PUT  /api/v1/auth/change-password   # 修改密码
POST /api/v1/auth/send-email-code   # 发送邮箱验证码
POST /api/v1/auth/send-sms-code     # 发送短信验证码
GET  /api/v1/auth/captcha           # 获取图形验证码
GET  /api/v1/auth/check-username    # 检查用户名是否可用
GET  /api/v1/auth/check-email       # 检查邮箱是否已注册
GET  /api/v1/auth/github            # GitHub登录跳转
GET  /api/v1/auth/github/callback   # GitHub登录回调
GET  /api/v1/auth/wechat            # 微信登录跳转
GET  /api/v1/auth/wechat/callback   # 微信登录回调
```

## 验收标准
- 注册登录流程完整无断点
- 表单校验信息明确，错误提示友好
- Token过期自动刷新，用户无感知
- 路由守卫正确拦截未授权访问
- 第三方登录流程顺畅

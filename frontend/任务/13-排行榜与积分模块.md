# 模块13：排行榜与积分模块

> 重要：本模块的接口路径必须与后端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
用户积分体系和排行榜系统，激励用户活跃参与社区，包括积分规则、等级体系、排行榜展示等。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 排行榜视觉效果要有竞争感和荣誉感
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 积分系统
- 积分获取规则
  - 每日登录：+5分
  - 发布帖子：+10分
  - 发布博客：+20分
  - 回答问题：+10分
  - 回答被采纳：+50分
  - 获得点赞：+2分
  - 分享项目：+15分
- 积分消耗
  - 悬赏提问
  - 兑换特权
- 积分明细记录

### 2. 等级体系
- 根据积分划分等级（如：新手、入门、进阶、高手、大师、宗师）
- 等级图标和颜色标识
- 等级对应权限（如高级别用户可以审核内容）

### 3. 排行榜
- 总积分排行
- 周/月活跃排行
- 贡献排行（回答数、采纳数）
- 创作排行（博客数、帖子数）
- 项目排行（星标数）
- 新星排行（新用户活跃度）
- 排行榜时间范围切换（本周/本月/全部）

### 4. 成就系统
- 成就徽章定义
  - 首次发帖、首次回答、首次博客等
  - 获赞达到特定数量
  - 连续登录天数
  - 帖子/博客数量里程碑
- 成就解锁通知
- 个人成就展示墙

## 接口对接
```
GET /api/v1/rankings?type=total&period=week&page=1  # 排行榜
GET /api/v1/users/:id/points                         # 积分明细
GET /api/v1/users/:id/level                          # 等级信息
GET /api/v1/users/:id/achievements                   # 成就列表
GET /api/v1/points/rules                             # 积分规则
```

- 前三名有特殊视觉展示
- 积分变动实时反映

- 等级图标在全站统一展示

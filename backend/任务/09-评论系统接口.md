# 模块09：评论系统接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现通用评论系统的后端接口，支持多目标类型、嵌套回复、提及用户等。

## 接口设计

```
GET    /api/v1/comments                     # 评论列表
POST   /api/v1/comments                     # 发表评论（需认证）
PUT    /api/v1/comments/:id                 # 编辑评论（需认证）
DELETE /api/v1/comments/:id                 # 删除评论（需认证）
POST   /api/v1/comments/:id/like            # 点赞评论
POST   /api/v1/comments/:id/report          # 举报评论
POST   /api/v1/comments/:id/pin             # 置顶评论（作者/管理员）
GET    /api/v1/comments/:id/replies         # 获取评论的回复
```

## 查询参数
```
GET /api/v1/comments?
  targetType=post&          # 目标类型
  targetId=xxx&             # 目标ID
  sort=latest|hot&          # 排序
  page=1&
  limit=20
```

## 业务逻辑
- 通用设计：targetType支持post/blog/project/snippet/question/answer
- 嵌套回复：最多3层嵌套，超过3层归并到第3层
- @提及：解析评论中的@username，生成通知
- 评论计数：评论发表/删除时更新目标内容的commentCount
- 防刷：同一用户对同一目标评论间隔至少10秒
- 内容过滤：敏感词检测

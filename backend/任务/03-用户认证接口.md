# 模块03：用户认证接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现完整的用户认证后端逻辑，包括注册、登录、Token管理、第三方登录、密码管理等。

## 功能清单

### 1. 注册
- 邮箱注册（邮箱验证 + 用户名唯一性检查 + 密码加密存储）
- 手机号注册（验证码验证）
- 图形验证码生成和校验
- 注册后自动发送欢迎邮件

### 2. 登录
- 邮箱/用户名 + 密码登录
- 手机号 + 验证码登录
- 登录失败次数限制和锁定
- 登录成功生成Access Token + Refresh Token

### 3. Token管理
- JWT Access Token（短期，如2小时）
- JWT Refresh Token（长期，如7天）
- Token刷新接口
- Token黑名单（Redis存储，退出登录后失效）

### 4. 第三方登录
- GitHub OAuth2.0
- 微信扫码登录
- QQ登录
- 首次第三方登录自动创建账号
- 已有账号绑定第三方

### 5. 密码管理
- 忘记密码（发送重置邮件，含重置链接Token）
- 重置密码
- 修改密码（需验证旧密码）
- 密码强度校验

### 6. 验证码服务
- 邮箱验证码发送
- 短信验证码发送（预留接口）
- 验证码存储（Redis，5分钟过期）
- 发送频率限制

## 接口设计
```
POST   /api/v1/auth/register            # 注册
POST   /api/v1/auth/login               # 登录
POST   /api/v1/auth/logout              # 退出
POST   /api/v1/auth/refresh-token       # 刷新Token
POST   /api/v1/auth/forgot-password     # 忘记密码
POST   /api/v1/auth/reset-password      # 重置密码
PUT    /api/v1/auth/change-password     # 修改密码
POST   /api/v1/auth/send-email-code     # 发送邮箱验证码
POST   /api/v1/auth/send-sms-code       # 发送短信验证码
GET    /api/v1/auth/captcha             # 获取图形验证码
GET    /api/v1/auth/github              # GitHub登录跳转
GET    /api/v1/auth/github/callback     # GitHub回调
GET    /api/v1/auth/wechat              # 微信登录
GET    /api/v1/auth/wechat/callback     # 微信回调
GET    /api/v1/auth/check-username      # 检查用户名是否可用
GET    /api/v1/auth/check-email         # 检查邮箱是否已注册
```

## 安全措施
- 密码使用bcrypt加密（salt rounds: 12）
- Token使用httpOnly cookie传输（可选）
- 登录接口rate-limit限流
- 验证码防暴力破解
- SQL注入/NoSQL注入防护
- XSS防护

## 验收标准
- 注册登录全流程正常
- Token刷新无感知
- 第三方登录OAuth流程完整
- 密码重置邮件正确发送
- 安全措施全部生效
- 异常情况返回友好错误信息

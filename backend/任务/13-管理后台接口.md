# 模块13：管理后台接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现管理后台所有功能的后端接口，所有接口需要管理员权限认证。

## 权限中间件
- 所有/api/v1/admin/*接口需要adminAuth中间件
- 验证用户角色为admin或moderator
- 记录所有管理操作到操作日志

## 接口设计

### 仪表盘
```
GET /api/v1/admin/dashboard/stats          # 概览统计
GET /api/v1/admin/dashboard/trends         # 趋势数据
GET /api/v1/admin/dashboard/activity       # 活跃度数据
```

### 用户管理
```
GET    /api/v1/admin/users                 # 用户列表
GET    /api/v1/admin/users/:id             # 用户详情
PUT    /api/v1/admin/users/:id             # 编辑用户
POST   /api/v1/admin/users/:id/ban         # 封禁
POST   /api/v1/admin/users/:id/unban       # 解封
POST   /api/v1/admin/users/:id/mute        # 禁言
POST   /api/v1/admin/users/:id/unmute      # 解除禁言
PUT    /api/v1/admin/users/:id/role        # 修改角色
POST   /api/v1/admin/users/:id/reset-pwd   # 重置密码
PUT    /api/v1/admin/users/:id/points      # 调整积分
GET    /api/v1/admin/roles                 # 角色列表
PUT    /api/v1/admin/roles/:id             # 编辑角色
```

### 内容审核
```
GET    /api/v1/admin/audit/queue           # 审核队列
POST   /api/v1/admin/audit/:id/approve     # 通过
POST   /api/v1/admin/audit/:id/reject      # 拒绝
POST   /api/v1/admin/audit/batch           # 批量审核
GET    /api/v1/admin/audit/history         # 审核历史
```

### 内容管理（帖子/博客/项目/代码/问答）
```
GET    /api/v1/admin/posts                 # 帖子管理列表
PUT    /api/v1/admin/posts/:id/pin         # 置顶
PUT    /api/v1/admin/posts/:id/feature     # 精华
PUT    /api/v1/admin/posts/:id/lock        # 锁定
DELETE /api/v1/admin/posts/:id             # 删除
POST   /api/v1/admin/posts/:id/restore     # 恢复

GET    /api/v1/admin/blogs                 # 博客管理
PUT    /api/v1/admin/blogs/:id/recommend   # 推荐
DELETE /api/v1/admin/blogs/:id             # 删除

GET    /api/v1/admin/projects              # 项目管理
DELETE /api/v1/admin/projects/:id          # 删除

GET    /api/v1/admin/questions             # 问答管理
PUT    /api/v1/admin/questions/:id/close   # 关闭
DELETE /api/v1/admin/questions/:id         # 删除
```

### 评论与举报
```
GET    /api/v1/admin/comments              # 评论列表
DELETE /api/v1/admin/comments/:id          # 删除评论
GET    /api/v1/admin/reports               # 举报列表
PUT    /api/v1/admin/reports/:id/handle    # 处理举报
GET    /api/v1/admin/reports/stats         # 举报统计
```

### 标签与分类
```
GET    /api/v1/admin/tags                  # 标签列表
POST   /api/v1/admin/tags                  # 创建标签
PUT    /api/v1/admin/tags/:id              # 编辑标签
DELETE /api/v1/admin/tags/:id              # 删除标签
POST   /api/v1/admin/tags/merge            # 合并标签
GET    /api/v1/admin/categories            # 分类列表
POST   /api/v1/admin/categories            # 创建分类
PUT    /api/v1/admin/categories/:id        # 编辑分类
DELETE /api/v1/admin/categories/:id        # 删除分类
```

### 敏感词管理
```
GET    /api/v1/admin/sensitive-words       # 敏感词列表
POST   /api/v1/admin/sensitive-words       # 添加
DELETE /api/v1/admin/sensitive-words/:id   # 删除
POST   /api/v1/admin/sensitive-words/import # 批量导入
```

### 运营工具
```
GET    /api/v1/admin/announcements         # 公告列表
POST   /api/v1/admin/announcements         # 发布公告
PUT    /api/v1/admin/announcements/:id     # 编辑
DELETE /api/v1/admin/announcements/:id     # 删除
GET    /api/v1/admin/banners               # Banner列表
POST   /api/v1/admin/banners               # 添加
PUT    /api/v1/admin/banners/:id           # 编辑
DELETE /api/v1/admin/banners/:id           # 删除
GET    /api/v1/admin/poll/config           # 获取论坛投票配置
PUT    /api/v1/admin/poll/config           # 更新论坛投票配置（标题/描述/开关/选项）
```

### 论坛投票（用户端）
```
GET  /api/v1/poll                          # 获取当前投票配置与票数（含当前IP是否已投票）
POST /api/v1/poll/vote                     # 提交投票（每个IP仅一次）
```

### 数据统计
```
GET /api/v1/admin/analytics/users          # 用户分析
GET /api/v1/admin/analytics/content        # 内容分析
GET /api/v1/admin/analytics/interactions   # 互动分析
GET /api/v1/admin/analytics/traffic        # 流量分析
GET /api/v1/admin/analytics/private-messages # 私信监控
POST /api/v1/admin/analytics/export        # 导出报表
```

### 系统配置
```
GET  /api/v1/admin/settings/:group         # 获取配置
PUT  /api/v1/admin/settings/:group         # 更新配置
POST /api/v1/admin/settings/email/test     # 测试邮件
```

### 操作日志
```
GET  /api/v1/admin/logs                    # 操作日志列表
POST /api/v1/admin/logs/export             # 导出日志
```

### 黑名单
```
GET    /api/v1/admin/blacklist             # 黑名单列表
POST   /api/v1/admin/blacklist             # 添加
DELETE /api/v1/admin/blacklist/:id         # 移除
```

## 验收标准
- 所有管理接口需要管理员权限
- 操作日志完整记录
- 批量操作功能稳定
- 数据统计查询性能良好（使用MongoDB聚合管道）
- 配置修改即时生效
- 敏感操作有二次确认机制

# 模块11：系统配置模块

> 重要：本模块的接口路径必须与后端管理后台接口（模块13）保持完全一致，任何变更需前后端同步更新。

## 模块概述
管理系统级别的配置项，包括站点信息、功能开关、安全设置等。

## 设计要求
- 科技蓝配色方案（坚决不使用紫色）
- 调用 UI/UX PRO MAX skills 设计
- 配置项分组清晰，修改即时生效
- 禁止使用任何表情包和emoji

## 功能清单

### 1. 站点基本设置
- 站点名称、Logo、favicon
- 站点描述、关键词（SEO）
- 备案号
- 联系邮箱
- 社交媒体链接

### 2. 功能开关
- 注册开关（开启/关闭注册）
- 邀请码注册（是否启用）
- 评论审核开关
- 帖子审核开关
- 第三方登录开关（GitHub/微信/QQ各自开关）
- 悬赏功能开关
- 积分功能开关

### 3. 内容设置
- 帖子最大字数
- 评论最大字数
- 图片上传最大尺寸
- 允许上传的文件类型
- 发帖间隔限制
- 评论间隔限制

### 4. 安全设置
- JWT过期时间
- 登录失败锁定次数
- 密码强度要求
- IP白名单/黑名单
- API限流配置

### 5. 邮件配置
- SMTP服务器设置
- 发件人配置
- 邮件模板管理
- 发送测试邮件

### 6. 存储配置
- 文件存储方式（本地/OSS/七牛）
- 存储密钥配置
- CDN配置

## 组件结构
```
src/pages/Settings/
├── SiteSettings.tsx           # 站点设置
├── FeatureSettings.tsx        # 功能开关
├── ContentSettings.tsx        # 内容设置
├── SecuritySettings.tsx       # 安全设置
├── EmailSettings.tsx          # 邮件配置
├── StorageSettings.tsx        # 存储配置
└── components/
    ├── SettingsForm.tsx        # 设置表单通用组件
    ├── ToggleSwitch.tsx        # 开关组件
    └── TestButton.tsx          # 测试按钮
```

## 接口对接
```
GET  /api/v1/admin/settings/:group        # 获取配置组
PUT  /api/v1/admin/settings/:group        # 更新配置组
POST /api/v1/admin/settings/email/test    # 测试邮件
POST /api/v1/admin/settings/storage/test  # 测试存储连接
```

## 验收标准
- 配置修改即时生效
- 敏感配置有确认弹窗
- 测试功能正常工作
- 配置项有清晰的说明文字
- 错误配置有友好提示

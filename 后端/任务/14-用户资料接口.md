# 模块14：用户资料接口

> 重要：本模块接口路径必须与前端模块08（个人中心）保持完全一致，任何变更需同步更新。

## 模块概述
实现面向普通用户的个人资料、社交关系、内容聚合等接口。区别于模块03（用户认证接口），本模块专注于用户资料和社交功能。

## 接口设计

### 1. 用户资料
```
GET    /api/v1/users/:id                   # 获取用户公开信息
PUT    /api/v1/users/profile               # 更新个人资料（需认证）
PUT    /api/v1/users/settings              # 更新设置（需认证）
POST   /api/v1/users/avatar               # 上传头像（需认证，调用上传服务）
POST   /api/v1/users/cover                # 上传封面图（需认证）
```

### 2. 用户内容聚合
```
GET    /api/v1/users/:id/posts             # 用户帖子列表
GET    /api/v1/users/:id/blogs             # 用户博客列表
GET    /api/v1/users/:id/projects          # 用户项目列表
GET    /api/v1/users/:id/snippets          # 用户代码分享列表
GET    /api/v1/users/:id/answers           # 用户回答列表
```

### 3. 社交关系
```
GET    /api/v1/users/:id/followers         # 粉丝列表
GET    /api/v1/users/:id/following         # 关注列表
POST   /api/v1/users/:id/follow            # 关注用户（需认证）
DELETE /api/v1/users/:id/follow            # 取消关注（需认证）
```

### 4. 收藏与历史
```
GET    /api/v1/users/:id/favorites         # 收藏列表
POST   /api/v1/users/favorites/folders     # 创建收藏夹（需认证）
PUT    /api/v1/users/favorites/folders/:id # 编辑收藏夹
DELETE /api/v1/users/favorites/folders/:id # 删除收藏夹
```

### 5. 贡献与成就
```
GET    /api/v1/users/:id/contributions     # 贡献热力图数据
GET    /api/v1/users/:id/achievements      # 成就列表
```

## 业务逻辑
- 获取用户信息时，根据隐私设置过滤敏感字段
- 关注/取关操作同步更新双方的计数字段
- 关注操作触发通知（调用通知服务）
- 内容聚合接口支持分页和排序参数
- 贡献热力图基于用户所有类型的内容发布时间统计

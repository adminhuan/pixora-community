# 模块06：问答系统接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现技术问答系统的后端接口，包括提问、回答、投票、采纳、悬赏等功能。

## 接口设计

```
# 问题
GET    /api/v1/questions                    # 问题列表
GET    /api/v1/questions/:id                # 问题详情
POST   /api/v1/questions                    # 创建问题（需认证）
PUT    /api/v1/questions/:id                # 编辑问题（需认证）
DELETE /api/v1/questions/:id                # 删除问题（需认证）
POST   /api/v1/questions/:id/vote           # 投票（+1/-1）
POST   /api/v1/questions/:id/follow         # 关注问题
POST   /api/v1/questions/:id/bounty         # 设置悬赏（需认证）
GET    /api/v1/questions/similar             # 查询相似问题

# 回答
GET    /api/v1/questions/:id/answers        # 问题的回答列表
POST   /api/v1/questions/:id/answers        # 提交回答（需认证）
PUT    /api/v1/answers/:id                  # 编辑回答
DELETE /api/v1/answers/:id                  # 删除回答
POST   /api/v1/answers/:id/vote             # 回答投票
POST   /api/v1/answers/:id/accept           # 采纳最佳答案（仅提问者）
```

## 业务逻辑
- 相似问题检测：基于标题关键词匹配已有问题
- 投票系统：每用户每目标只能投一票，可切换赞同/反对
- 采纳最佳答案：自动标记问题为已解决，给回答者积分奖励
- 悬赏系统：
  - 扣除提问者积分
  - 设置过期时间
  - 到期未采纳自动分配给最高票回答
  - 无回答则退还积分
- 投票积分：回答获赞+2分，获反对-1分

## 验收标准
- 投票逻辑正确（不能重复投票，可切换）
- 采纳操作权限正确（仅提问者可采纳）
- 悬赏积分扣除和结算正确
- 相似问题推荐有效
- 关注问题后有新回答时触发通知

# 模块10：消息通知接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现消息通知系统的后端逻辑，包括通知生成、WebSocket实时推送、通知管理等。

## 接口设计

```
# REST API
GET    /api/v1/notifications                # 通知列表（需认证）
GET    /api/v1/notifications/unread-count    # 未读数量
PUT    /api/v1/notifications/:id/read        # 标记已读
PUT    /api/v1/notifications/read-all        # 全部标记已读
DELETE /api/v1/notifications/:id             # 删除通知
PUT    /api/v1/notifications/settings        # 更新通知设置
GET    /api/v1/notifications/settings        # 获取通知设置

# 私信 API（需认证）
GET    /api/v1/messages/conversations              # 会话列表
POST   /api/v1/messages/conversations/open         # 按用户打开/创建会话
GET    /api/v1/messages/conversations/:id/messages # 会话消息列表
POST   /api/v1/messages/send                       # 发送私信（支持文本+attachments）
POST   /api/v1/messages/:id/recall                # 撤回消息（2分钟内）
PUT    /api/v1/messages/conversations/:id/read     # 会话全部标记已读
GET    /api/v1/messages/unread-count               # 私信未读数量
GET    /api/v1/messages/block/:userId/status       # 获取与目标用户屏蔽状态
POST   /api/v1/messages/block/:userId              # 屏蔽用户
DELETE /api/v1/messages/block/:userId              # 取消屏蔽

# 上传 API（需认证）
POST   /api/v1/upload/image                        # 上传图片附件
POST   /api/v1/upload/file                         # 上传文件附件
```

## WebSocket事件

```
# 服务端 -> 客户端
notification:new          # 推送新通知
notification:count        # 推送未读数量更新
private-message:new       # 推送私信内容
private-message:count     # 推送私信未读数量更新
private-message:typing    # 推送正在输入状态
private-message:recalled  # 推送消息撤回状态

# 客户端 -> 服务端
notification:read         # 标记已读
notification:connect      # 建立连接（携带Token）
private-message:typing    # 上报正在输入状态
```

## 通知生成规则
| 触发动作 | 通知类型 | 接收人 |
|---------|---------|-------|
| 评论帖子/博客/项目 | comment | 内容作者 |
| 回复评论 | reply | 被回复者 |
| @提及 | mention | 被@用户 |
| 点赞内容 | like | 内容作者 |
| 关注用户 | follow | 被关注者 |
| 回答问题 | answer | 提问者 |
| 采纳回答 | accept | 回答者 |
| 关注问题有新回答 | question_update | 关注者 |
| 系统公告 | system | 全部用户 |
| 发送私信 | private_message | 接收私信用户 |

## 业务逻辑
- 通知生成：各业务模块调用通知服务统一生成
- 实时推送：通过Socket.IO推送到在线用户
- 邮件通知：根据用户设置，重要通知发送邮件
- 通知合并：短时间内同类型通知合并（如"xxx等5人赞了你的帖子"）
- 在线状态管理：Redis维护用户在线状态
- 私信发送接口增加限流（每分钟 30 次）防滥用
- 私信发送接入敏感词检查，命中后返回 `CONTENT_SENSITIVE`
- 私信列表单次分页上限 50 条，默认 30 条
- 私信支持 `attachments` 字段（图片/文件），文本可为空但内容与附件不能同时为空
- 私信发送前增加双向屏蔽校验：自己屏蔽对方或被对方屏蔽都禁止发送

## 验收标准
- 各类通知正确生成
- WebSocket实时推送稳定
- 未读计数准确
- 通知合并逻辑正确
- 邮件通知按设置发送
- 断线重连后不丢失通知

# 模块11：搜索服务接口

> 重要：本模块接口路径必须与前端对应模块保持完全一致，任何变更需前后端同步更新。

## 模块概述
实现全站搜索功能的后端接口，支持多类型内容搜索、搜索建议、热门搜索等。

## 接口设计

```
GET /api/v1/search                      # 全站搜索
GET /api/v1/search/suggestions          # 搜索建议/自动补全
GET /api/v1/search/hot                  # 热门搜索词
```

## 查询参数
```
GET /api/v1/search?
  q=keyword&                # 搜索关键词
  type=all|post|question|blog|snippet|project|user&  # 内容类型
  tag=xxx&                  # 标签筛选
  author=xxx&               # 作者筛选
  dateFrom=2024-01-01&      # 时间范围起始
  dateTo=2024-12-31&        # 时间范围结束
  sort=relevance|latest|popular&  # 排序
  page=1&
  limit=20
```

## 技术方案

### 初期方案：MongoDB文本搜索
- 使用MongoDB text index
- 支持中文分词
- 简单高效，无需额外服务

### 进阶方案：Elasticsearch（预留）
- 更好的搜索体验和性能
- 支持高亮、模糊匹配、权重配置
- 需要数据同步机制

## 业务逻辑
- 搜索结果按相关度评分排序
- 搜索关键词高亮标记（返回匹配片段）
- 搜索建议：基于已有内容标题的前缀匹配
- 热门搜索：基于搜索日志统计，Redis缓存
- 搜索日志记录（用于统计和优化）
- 搜索结果缓存（Redis，5分钟过期）

## 验收标准
- 搜索结果准确，支持中文
- 搜索建议响应时间不超过200ms
- 热门搜索词更新及时
- 多类型搜索结果聚合正确
- 搜索无结果时有建议

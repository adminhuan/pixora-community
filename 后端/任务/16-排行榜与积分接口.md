# 模块16：排行榜与积分接口

> 重要：本模块接口路径必须与前端模块13（排行榜与积分）保持完全一致，任何变更需同步更新。

## 模块概述
实现面向普通用户的排行榜查询、积分明细查看、等级信息、成就系统等接口。

## 接口设计

### 1. 排行榜接口
```
GET    /api/v1/rankings                    # 排行榜数据
```

### 2. 积分接口
```
GET    /api/v1/users/:id/points            # 用户积分明细
GET    /api/v1/users/:id/level             # 用户等级信息
GET    /api/v1/points/rules                # 积分规则说明
```

### 3. 成就接口
```
GET    /api/v1/users/:id/achievements      # 用户成就列表
GET    /api/v1/achievements                # 全部成就定义
```

## 查询参数
```
GET /api/v1/rankings?
  type=total|contribution|creation|project|newstar&  # 排行类型
  period=week|month|all&                              # 时间范围
  page=1&
  limit=20

GET /api/v1/users/:id/points?
  type=earn|spend|all&          # 积分变动类型
  page=1&
  limit=20
```

## 业务逻辑
- 排行榜数据Redis缓存，每小时更新一次
- 总积分排行：按用户总积分降序
- 贡献排行：按回答数+采纳数降序
- 创作排行：按博客数+帖子数降序
- 项目排行：按项目星标总数降序
- 新星排行：注册30天内的用户按活跃度排序
- 等级计算规则：
  - 0-100分：新手
  - 101-500分：入门
  - 501-2000分：进阶
  - 2001-5000分：高手
  - 5001-20000分：大师
  - 20000分以上：宗师
- 积分变动由各业务模块调用积分服务统一处理
- 成就解锁在积分变动时检查条件并触发通知
